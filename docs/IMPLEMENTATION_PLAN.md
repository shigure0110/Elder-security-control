# Elder Security Control - 功能落地模块计划

## 目标
围绕老人购物与支付行为（支付宝、微信、抖音、拼多多、淘宝）建立“监测-风控-汇总-家属群通知”闭环。

## 模块拆分

### 1. 平台账单采集模块
- **职责**：采集各平台订单/支付流水（优先支付宝、微信）。
- **输入**：平台账单接口、消息通知解析、用户授权信息。
- **输出**：统一账单事件 `BillingEvent`。
- **优先级**：P0（支付宝、微信），P1（抖音/拼多多/淘宝）。

### 2. 账单标准化与去重模块
- **职责**：统一币种、金额单位、商户名称，处理重复通知。
- **输入**：`BillingEvent`。
- **输出**：`NormalizedBillingEvent`。
- **关键点**：同一交易号 + 平台 + 时间窗去重。

### 3. 风险规则增强模块
- **职责**：在现有关键词与支付规则基础上，新增“购物行为风险”规则。
- **示例规则**：
  - 深夜短时高频下单。
  - 新商户首次消费且金额异常。
  - 短时间内跨平台重复购买同类商品。
- **输出**：风险分、风险等级、解释原因。

### 4. 每日小额账单汇总模块
- **职责**：按日聚合“小额账单”，并生成家属可读摘要。
- **输入**：`NormalizedBillingEvent` + 配置阈值（如 ≤200 元）。
- **输出**：日报文本（总笔数、总金额、Top 商户、异常说明）。

### 5. 微信群路由与发布模块
- **职责**：将日报或告警发送到指定微信群。
- **能力**：
  - 配置群聊名称 + 群聊唯一 ID。
  - 多群策略（默认群、告警群、备份群）。
  - 失败重试与投递状态追踪。

### 6. 设置中心（前端）模块
- **职责**：提供可配置界面，支持：
  - 各平台管控开关。
  - 小额阈值与汇报时间。
  - 目标微信群配置。
- **状态**：已新增 Compose 设置页雏形，可继续接入持久化与后端同步。

### 7. 调度与任务执行模块
- **职责**：定时生成日报、失败补偿、离线重试。
- **建议实现**：Android 侧可先用 WorkManager；服务端可用 Cron/队列。

### 8. 家属可视化与审计模块
- **职责**：展示趋势、明细、告警命中原因与处理记录。
- **关键点**：支持“为什么发这条告警”的可解释性。

## 里程碑建议
- **M1（2 周）**：设置中心 + 支付宝/微信小额日报 + 固定微信群投递。
- **M2（2~4 周）**：风险规则增强 + 多群路由 + 失败重试。
- **M3（4 周+）**：多平台扩展 + 家属端可视化完善 + 数据审计能力。
